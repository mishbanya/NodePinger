# Тестовое задание

## Цель

Реализовать простое приложение с сетевым взаимодействием: подключение к P2P-ноде, получение данных по идентификатору (CID) и мониторинг доступности через пинг с отображением задержки.

## Задание

Нужно написать **Android-приложение** на **Kotlin**, которое:

1. **Подключается к заданной ноде** по указанному multiaddress (например, TCP или QUIC).
2. **Скачивает данные по CID**: пользователь вводит CID, приложение запрашивает блок/файл у ноды и выводит результат (содержимое или метаданные) в интерфейсе.
3. **Пингует ноду с заданным интервалом** (каждые **N** секунд, N — настраиваемый параметр, например 1–2 сек) и **выводит текущую latency** (задержку в мс) в интерфейсе.

### Требования к реализации

- **Платформа и язык**: **Android**, только **Kotlin** (без Java для бизнес-логики).
- **Библиотека для работы с нодой**: обязательно использовать **[Peergos Nabu](https://github.com/Peergos/nabu)** (минимальная Java-реализация IPFS/libp2p; подключается, в том числе, через JitPack).
- **Интерфейс**: достаточно простого UI (кнопки, поле ввода CID, область для вывода результата и отображения latency). Дизайн не оценивается приоритетно.
- **Обработка ошибок**: при недоступности ноды, таймауте или неверном CID приложение не должно падать; желательно показывать пользователю понятное сообщение об ошибке.
